# OctoAcme Project Management Processes

Welcome to the OctoAcme project management documentation. This folder contains comprehensive guidance for running projects using OctoAcme's proven processes.

## Quick Overview

OctoAcme uses a structured, iterative project management approach grounded in customer value and data-driven decision-making. The lifecycle spans five distinct phases:

1. **Initiation** — Define the problem and secure stakeholder alignment
2. **Planning** — Break work into shippable increments with clear acceptance criteria
3. **Execution** — Daily standups and iterative delivery with quality gates
4. **Release** — Standardized deployment with rollback plans
5. **Close & Retrospective** — Capture learnings for continuous improvement

This end-to-end framework ensures projects are validated before they begin and that learnings are systematically captured to improve future delivery.

## Core Principles

- **Customer-first**: Prioritize customer value and usability
- **Iterative delivery**: Deliver small, testable increments
- **Clear ownership**: Each project has a named Project Manager (PM) and Product Lead
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback and learning

## Key Roles

- **Project Manager (PM)**: Coordinates delivery, schedules, risks, and communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes the backlog, and measures success
- **Developers**: Implement features while collaborating on design and testing
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs and approvals

## Communication & Cadence

OctoAcme emphasizes clear, consistent communication through:
- **Daily standups** (15 min) — Progress, blockers, and dependencies
- **Weekly PM-PdM syncs** — Strategic alignment and risk review
- **Twice-weekly delivery team standups** — Status updates and sprint progress
- **Monthly stakeholder updates** — High-level progress and outcomes

## Quality & Execution

Quality is embedded throughout delivery:
- **Prioritized backlog** with acceptance criteria
- **Definition of Done** checklists
- **Small PRs** (≤400 lines) with mandatory CI testing and peer review
- **Unit, integration, and end-to-end smoke testing**
- **Standardized release checklists** including staging verification and rollback planning
- **Continuous risk management** with a maintained risk register

## Continuous Improvement

OctoAcme institutionalizes learning through:
- **Sprint/milestone retrospectives** focused on "what went well," "what could improve," and actionable next steps
- **Assigned owners and due dates** for improvements
- **Psychological safety** culture encouraging feedback
- **Versioned documentation** that captures decisions and learnings
